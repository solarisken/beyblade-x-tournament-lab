# Quick Start

1. Open the app through GitHub Pages or a local web server.
2. Let **Auto-Pilot** complete the fast search and scenario lab.
3. Review the deck, the weakest backup matchup, and the critical floor.
4. Open **Adaptive Tests** and run only the highest-priority short batch.
5. Record each result with its finish type. Position alternates automatically.
6. After the batch commits, allow Auto-Pilot to rebuild the recommendation.
7. Stop testing a matchup once the coach deprioritizes it.
8. Run **Deep exhaustive audit** on a desktop when a complete allocation audit is required.

Do not interpret the scenario match rate as physical proof. Use the evidence status and battle ledger for that determination.
